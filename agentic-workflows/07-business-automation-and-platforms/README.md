# Business Automation and Agent Platforms

**Category:** Optional platform extension

## Intent

Learn the software stack that turns agent ideas into client-facing automation systems: orchestration tools, visual agent builders, and the surrounding systems that make the work operational.

## Why this stage exists

The first six stages teach how agentic systems work. This stage teaches how people often **ship** them in the real market:

- with orchestration platforms like **n8n**, **Make**, and **Zapier**
- with visual AI builders like **Flowise**, **Langflow**, and **Dify**
- with supporting systems like **Supabase**, **Airtable**, **Notion**, **HubSpot**, **Slack**, and **Stripe**

If you want to build a business around automation and agent delivery, you need more than model knowledge. You need platform judgment.

## What this stage trains

- picking the right tool for a client instead of forcing your favorite platform
- seeing the difference between a fast demo tool and a durable delivery platform
- using visual tools without losing reliability, observability, or control
- packaging software choices into a repeatable service offering

## How to study this stage

Do not treat this stage like a binge of tutorials. The useful unit is not "I watched a video." The useful unit is "I can now build and defend one real workflow."

For each topic:

1. read the README
2. run the demo
3. do the homework
4. clone one real use case you could sell or deploy
5. write down where the platform helped and where it fought you

## Common mistakes

- learning one platform so deeply that every client problem starts looking like the same tool
- shipping visual flows with no naming conventions, no failure handling, and no observability
- treating hosted convenience as free when lock-in and margins are ignored
- using agent builders where normal automation would be simpler and more reliable
- forgetting that the surrounding systems usually matter as much as the orchestration layer
