# Homework — Levels and Expectations

## Exercise 1: Level self-assessment

For the level you currently hold (or closest to where you are), answer these three questions honestly and specifically:

1. **Scope**: what is the largest thing you currently own end-to-end? Is it a function, a module, a service, a domain? Be specific.
2. **Autonomy**: what is the last thing you decided without asking anyone? What is the last thing you should have decided yourself but asked about instead?
3. **Judgment**: describe one technical call you made in the last month without a playbook. What was the situation, what did you decide, and how did it turn out?

## Exercise 2: Next-level gap analysis

For the level above where you are now, read the description in the README and write down:

1. **Three things the next level does that you are not doing yet.** Be concrete — not "have more autonomy" but "identify and propose new work items rather than only executing assigned ones."
2. **What would need to be true about your work this quarter** for someone to have evidence that you are already operating at the next level?
3. **What is your actual biggest gap?** Not the one that sounds impressive to say — the one that is actually limiting you.

## Exercise 3: Evidence inventory

Make a list of the last five significant things you did at work. For each one, answer:

1. Was this visible to people who would be making a promotion decision?
2. Did it generate a written artifact (design doc, post-mortem, thorough PR description) that could be referenced later?
3. What level does this piece of work demonstrate — and is that the level you're targeting?

If most of your work in the last quarter was invisible (in PRs that got merged with no comment, in Slack threads that scrolled away, in verbal conversations), you have an evidence problem regardless of the quality of the work.

## Reflection question

Write one paragraph on: "What is one concrete thing I can start doing this week that is more consistent with the level above me?"

Do not make it aspirational. Make it specific and actionable enough that you could report back next week on whether you did it.
