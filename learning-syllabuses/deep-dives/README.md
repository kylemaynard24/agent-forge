# Deep dives

Generated by the `/deep-dive` skill (in `.claude/skills/deep-dive/`). One file per external topic — for topics that have a canonical repo README (`agentic-workflows/`, `software-engineering/architecture/`, `software-engineering/design-patterns/`), deep dives are appended directly to that README instead of living here.

## Layout

```
deep-dives/
├── design-patterns/
│   └── headfirst-chN-<short>.md     ← Head First chapter deep dives
└── devops/
    └── <topic-slug>.md              ← devops topic deep dives (Bicep, Container Apps, etc.)
```

Each file accumulates deep-dive sections over time. New dives append; previous dives are never overwritten. Each section is dated and tagged with the focus area so a file's history reads as a record of the topics you've gone deep on.

## How to add to one

Run `/deep-dive <topic>` (or `/deep-dive <topic>: <focus area>`). The skill will create the file on first invocation, append on subsequent ones.

## Why a separate location

Repo READMEs (for `agentic-workflows`, `architecture`, `design-patterns`) are user-authored teaching content with a specific voice. Deep dives append in-place there because the user wants the original teaching plus the deeper exploration as one growing artifact.

External topics (devops, Head First chapters) don't have a canonical single README — the source material is the official Microsoft Learn / Azure docs / Docker docs / the *Head First* book itself. Deep dives for those topics live here so they accumulate in a discoverable location instead of bloating the syllabus or being scattered.
