# progress/ — sprint plans, daily slices, and personal state

Three layers:

## 1. Per-subject state — `<subject>/state.md`

Long-lived. Tracks your position in each subject's master syllabus (level, topic, current step). Tracked by git.

## 2. Sprints — `sprints/YYYY-MM-DD/`

Generated by `/next-sprint`. Each sprint covers one step (`read`, `demo`, or `implement`) for each of the four subjects, broken into ~10–14 items totaling ~10–15 hours of work. A sprint takes ~1–2 weeks of part-time effort.

Per-sprint files:
- `sprint.md` — full reading material + apply tasks + questions + extra credit
- `items.md` — flat checklist read by `/daily-tasks`

Tracked by git so the learning history accumulates and friends can see what's been covered.

## 3. Daily slices — `YYYY-MM-DD/`

Generated by `/daily-tasks`. Each daily folder has:
- `todo.md` — today's 2–3 items (~1–2 hours), pulled from the active sprint
- `working-folder/<subject>/` — scratch space for apply-task code

`todo.md` is tracked by git. `working-folder/` is **gitignored** (`progress/*/working-folder/` in `.gitignore`) — that's where messy experiments and any local secrets in scratch code stay local.

## Bootstrapping

If `state.md` files are missing (fresh clone), the skills auto-bootstrap from each syllabus's Level 1 Topic 1 / step `read`. Run `/next-sprint` first; then `/daily-tasks`.

## Resetting

If you want to start a state file over, you can safely `rm` it — the next `/next-sprint` will bootstrap a fresh starter from the syllabus.
