# progress/ — sprint state and daily todos

Layout:
- `<subject>/state.md` — your sprint position (level, topic, next step) for each tracked subject. Tracked by git.
- `YYYY-MM-DD/todo.md` — today's plan, generated by `/daily-tasks`. Tracked by git.
- `YYYY-MM-DD/working-folder/<subject>/` — scratch space for today's apply-task work. **Gitignored** (pattern `progress/*/working-folder/` in `.gitignore`).

State files and daily todos are committed so the learning journey is preserved. Apply-task scratch code stays local — that's where you can paste secrets, experiment with broken builds, or write throwaway code without thinking about what you're committing.

Shared curriculum lives in `learning-syllabuses/` at the repo root. The skill itself is in `.claude/skills/daily-tasks/`.

If you want to start a state file over, you can safely `rm` it — the next `/daily-tasks` run will bootstrap a fresh starter from the syllabus.
