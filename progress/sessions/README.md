# Sessions

Snapshots of working context from individual Claude sessions, generated by the `/save-session` skill (in `.claude/skills/save-session/`).

## Why these exist

Conversations end. You step away from a project for two weeks and lose context. A friend pulls the repo and wants to know what you've been working on. A new Claude session needs to bootstrap from scratch.

A session snapshot is a one-page summary of what was happening: the goal, what got done, what's in flight, what's next, open questions. It's not a transcript — it's a structured digest sized to be readable in 5 minutes.

## File naming

```
progress/sessions/YYYY-MM-DD-HHMM-<slug>.md
```

The timestamp is when the snapshot was saved (local time). The slug is a 3–6 word kebab-case description of the session's focus.

## How to use

- Run `/save-session` to snapshot the current session. The skill auto-detects focus from recent context, or accepts an explicit focus argument.
- A future Claude session (or you, days later) can read a snapshot to bootstrap context without re-reading the whole conversation.
- Snapshots are tracked by git by default — they accumulate as a record of how the project evolved over time.

## What snapshots are NOT

- Not a transcript of the conversation.
- Not a substitute for git commits (commits document changes; snapshots document thinking).
- Not for long-lived user facts (those belong in auto-memory at `~/.claude/projects/.../memory/`).
