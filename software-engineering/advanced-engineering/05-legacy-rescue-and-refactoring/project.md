# Project — Modernization Plan

## Goal

Take one medium-small legacy codebase or subsystem and produce a **credible modernization path** that improves it without fantasy.

## Deliverables

1. A map of the current dependency tangle
2. One implemented seam or adapter
3. One behavior-preserving refactor
4. A 3-step modernization roadmap
5. A risk and rollback section

## Constraints

- One roadmap step must be explicitly deferred because it is too risky now
- One improvement must make testing easier
- One improvement must reduce the need for team folklore

## Done when

- [ ] The plan reads like something a real team could approve
- [ ] The first move is small enough to ship
- [ ] The roadmap is honest about cost and risk
