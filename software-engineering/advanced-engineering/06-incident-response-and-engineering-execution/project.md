# Project — Operations and Communication Pack

## Goal

Create a full **incident-response pack** for one integrated project in this repo or one project of your own.

## Deliverables

1. A runbook for one critical failure mode
2. A simple dashboard or observability sketch
3. A severity matrix
4. A communication template for internal and external updates
5. A postmortem template and one filled-out example
6. One short ADR or design note for a follow-up improvement

## Constraints

- The runbook must start with detection and triage, not with "fix code"
- One section must define when to stop mitigation and escalate
- One section must explain what data is needed to declare recovery
- The ADR must compare at least two options

## Done when

- [ ] Another engineer could use the pack during a stressful hour
- [ ] Recovery criteria are explicit
- [ ] The follow-up writing improves the system, not just the paperwork
