# Challenges — Debugging Basics

> Practice checking values and narrowing down why code is behaving strangely with five short challenges.

These challenges are intentionally smaller than the main homework. They are meant to help a beginner get more repetitions without feeling overwhelmed.

## How to use these

1. Pick one challenge at a time.
2. Predict what your code should do before you run it.
3. Keep the script small enough to explain line by line.
4. If you get stuck, go back to the demo and change only one thing at a time.

## The five challenges

1. Write a tiny function with a bug on purpose, then fix it after printing the wrong result.
2. Add `console.log` statements to inspect the values going into a calculation.
3. Create a script that prints an unexpected answer, then explain what caused it.
4. Take a working script and change one value to predict how the output will change before running it.
5. Write a short note describing the difference between the symptom and the actual cause of a bug.

## Done when

- you completed at least three of the five without copying blindly
- you can explain your solution in plain language
- you can change one input or value and still predict the new output
