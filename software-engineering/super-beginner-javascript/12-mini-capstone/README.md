# Mini Capstone

**Track:** Super beginner JavaScript

## What this topic is about

Combine the beginner topics into one small program you can explain confidently.

## Why it matters

If someone is absolutely new to programming, mini capstone is one of the first places where code stops looking like random punctuation and starts looking like a set of ideas. This topic helps a learner see that programs are not magic; they are just instructions acting on values in predictable ways.

## When to use it

- you want to write code for building a console-based student tracker, task app, or mini shop
- you need one small building block before moving to larger programs
- you want to explain not just what the syntax is, but why it exists

## The core idea

Start simple:

1. name the thing you are working with
2. inspect what value it currently has
3. change it or use it deliberately
4. explain the result in plain language

That habit is more important than memorizing syntax.

## Common beginner mistakes

- copying code without predicting what it will print
- changing several things at once and losing track of the cause
- memorizing syntax before understanding the underlying idea

## Teaching questions and explanations

These questions are meant for teaching and self-checking. They help a beginner connect **Mini Capstone** to concrete situations instead of memorizing definitions without understanding.

### Scenario 1 — "A learner can run the code, but cannot explain what it is doing"

**Question:** If a learner is stuck in that state, is **Mini Capstone** the kind of topic you should slow down and revisit directly?

**Answer:** Yes.

**Explanation:** Beginner topics matter most when they let someone explain code in plain language. **Mini Capstone** is not just syntax to copy. It is part of the learner's mental model for how a program stores data, makes choices, repeats work, or organizes behavior.

**Why not skip ahead to Strings Numbers and Booleans or Conditionals:** later topics become much easier when the learner can already narrate this one confidently. Moving too fast usually creates fragile understanding.

### Scenario 2 — "Someone asks why this topic exists at all"

**Question:** How would you answer without using jargon?

**Answer:** I would explain that **Mini Capstone** helps a learner control one important part of how programs behave.

**Explanation:** A good beginner explanation should feel concrete: what problem does this topic solve, what kind of code becomes possible because of it, and what confusion disappears once it clicks? That is the real teaching goal.

**Why not teach it as a rule sheet:** because beginners retain ideas better when they see what the concept lets them do, not just what symbols they are supposed to type.
